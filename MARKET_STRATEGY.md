## Decision

DreamCleanr wins as `AI workflow hygiene for macOS`, not as a generic Mac cleaner.

This rollup stays subordinate to [DREAMCLEANR_MASTER_STRATEGY.md](DREAMCLEANR_MASTER_STRATEGY.md) and its `$dreamcleanr-strategy-analysis-operator` workflow.

The category wedge is:

- trust-first cleanup
- AI and developer workflow awareness
- local-first execution
- receipt-first transparency

## Competitive Position

DreamCleanr should position against four market lanes:

1. broad premium cleaners like CleanMyMac
2. focused utilities like DaisyDisk
3. power-user automation tools like Hazel
4. free maintenance utilities like OnyX

DreamCleanr does not win by becoming more generic than those products.
It wins by being more specific to Docker, AI tools, developer environments, and safe cleanup logic.

## Go-To-Market Summary

Priority channels:

1. GitHub discovery
2. developer and AI workflow communities
3. SEO for high-intent Mac and AI/dev cleanup queries
4. package-manager discoverability later

Weak channels for now:

- broad consumer paid acquisition
- ad-heavy funnels
- enterprise-first sales motion

## Message

Primary category:

> The Mac cleanup and optimization layer for AI and developer workflows.

Primary promise:

> Clean while you sleep, without breaking the tools you use during the day.

## Strategic Rule

Do not let public positioning drift back toward “generic cleaner.”

The current moat is:

- Docker + AI tool awareness
- protected Claude/Codex state
- local receipts
- GitHub-first trust

### Main business risks

- trust gap in the cleanup category
- install friction for non-terminal users
- unclear monetization timing
- over-expanding into too many platforms too early

### Main business advantage

Low hosting cost means DreamCleanr can grow patiently and profitably if it finds a durable niche with enough willingness to pay for convenience and trust.

## Implementation Plan

### Phase 1

- improve on-page SEO and structured data
- strengthen copy around the AI/developer niche
- keep the homepage onboarding demo explicit about sample behavior
- reduce install friction for Claude, Codex, and VS Code
- keep all public copy grounded in what ships today

### Phase 2

- ship macOS shell convenience layer
- test one premium packaging option
- publish comparison and FAQ content

### Phase 3

- evaluate team/admin offering only if demand appears
- defer any backend-heavy analytics or accounts until clearly justified

## Immediate Recommendations

1. Keep DreamCleanr positioned as `AI workflow hygiene for macOS`.
2. Keep core functionality free while product trust compounds.
3. Keep the site and pricing page explicit about `shipped now` versus `coming next`.
4. Invest next in distribution and convenience, not infrastructure.
5. Use usage, issue volume, and install demand to decide when pricing should become more assertive.
