# DreamCleanr Investor Pitch Deck

This is the markdown draft for a future presentation deck. It is grounded in current product truth and leaves traction placeholders blank where data does not exist yet.

## 1. Title

DreamCleanr  
`AI workflow hygiene for macOS`

Positioning statement:

> DreamCleanr is the Mac optimization layer built specifically for AI developers, helping reclaim storage, protect dev environments, and keep local workflows fast.

## 2. Problem

- AI-heavy and developer-heavy Macs accumulate Docker sprawl, cache growth, stale helpers, and broken disk headroom
- generic cleaners do not understand active developer state
- manual cleanup is slow, risky, and hard to trust

## 3. Solution

DreamCleanr is a local-first cleanup and optimization layer for AI and developer workflows on macOS.

Today it already offers:

- Docker-aware cleanup
- Claude/Codex-aware protection
- safe cache cleanup
- receipts
- scheduling
- MCP access

## 4. Market

- millions of developers rely on macOS
- AI coding tools and local model workflows are increasing machine pressure
- the existing cleaner market is broad, but not dev-native

## 5. Product Demo

- scan
- classify reclaimable versus protected state
- clean safely
- leave a visible receipt

Demo notes:

- current site demo is sample-only
- real premium onboarding arrives later in the macOS shell

## 6. Business Model

- `Community`: free CLI, MCP, receipts, and scheduling
- `Pro`: later one-time premium macOS shell
- `Team`: later pilot for policy and rollout workflows

## 7. Traction

Fill in when available:

- GitHub-first distribution

## 9. Competition

- CleanMyMac: broad and premium, not dev-specific

- Hazel / manual scripts: powerful but fragmented

DreamCleanr advantage:

- AI/dev workflow awareness
- receipt-first trust
- local-first architecture
- premium-shell expansion path
