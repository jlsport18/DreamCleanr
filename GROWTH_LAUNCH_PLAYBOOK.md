# DreamCleanr Growth And Launch Playbook

This is the grounded launch playbook for DreamCleanr. It assumes the product is real, useful, and still early.

Related docs:

- [ONBOARDING_CONVERSION_SPEC.md](ONBOARDING_CONVERSION_SPEC.md)
- [CREATOR_OUTREACH.md](CREATOR_OUTREACH.md)
- [site/pricing.html](site/pricing.html)

## Core launch message

DreamCleanr is the cleanup and optimization layer for AI and developer-heavy Macs.

Public launch language must stay honest:

- say what ships today
- say what is coming next
- never present future Python, Node, Ollama, LM Studio, or Hugging Face cleanup intelligence as already available

## Site-demo rule

- the homepage onboarding flow is a truthful demo, not a live browser scan
- keep sample numbers labeled as sample numbers
- reserve the real install -> scan -> upgrade path for the future macOS shell

## Primary channels

### Product Hunt

- Position DreamCleanr as `trust-first cleanup for AI developer Macs`
- Lead with before/after receipt visuals, not hype
- Use the repo, receipts, and install flow as proof of legitimacy

### Hacker News

- Use a technical `Show HN` angle:
  - why generic cleaners fail developers
  - how DreamCleanr protects active AI state
  - why preview-first and receipt-first matter

### Indie Hackers and X

- Share quantified cleanup wins
- Show actual product boundaries and current roadmap
- Avoid fake “AI magic” framing

## Creator outreach

Use [CREATOR_OUTREACH.md](CREATOR_OUTREACH.md) for:

- DM scripts
- creator-angle mapping
- incentive rules
- follow-up cadence

### Focused Reddit communities

- Keep posts transparent and useful
- Likely relevant audiences:
  - `r/LocalLLaMA`
  - `r/macapps`
  - `r/MachineLearning`
  - developer/macOS productivity communities where self-promo rules allow it

## Launch assets to prepare

- before/after cleanup receipt visual
- CLI scan and cleanup screenshots
- MCP setup screenshots for Claude, Codex, and VS Code
- future disk-category preview mockups only if clearly labeled as upcoming
- short animated install/update clip for the site

## Response guidelines

- answer every serious comment with technical depth
- acknowledge current limitations quickly
- explain what is shipped versus planned
- keep the tone calm, sharp, and developer-native
- never use fabricated user quotes or inflated performance claims

### Launch-day comment strategy

- reply to every serious launch comment
- lead with the exact current product behavior
- acknowledge planned detectors as planned
- keep humor light and technical, never evasive

## Pricing and CTA guidance

- `Community` is live and free
- `Pro` is anchored as `coming soon`
- `Team` is a later pilot
- current CTA ladder:
  - `Download Free`
  - `See What It Cleans`
  - `Join Pro Early Access`
  - `Follow Updates on GitHub`

## Incentive policy

Allowed:

- early access
- future lifetime Pro for a small number of high-signal testers
- later affiliate exploration only after the commercial model is real

Not allowed:

- paid-positive coverage
- fake urgency
- off-platform waitlist tooling in this phase

## GitHub-first collection model

Use GitHub-native mechanisms before adding external growth tooling:

- releases
- stars and watchers
- issues labeled for install friction or feature demand
- Discussions or the `pro-interest.yml` issue template for Pro interest

Do not add email capture or backend growth tooling in this phase unless GitHub-native interest proves insufficient.

## Later growth loops

These are later features, not current promises:

- shareable receipt summary image
- safe “you reclaimed X GB” card
- native shell screenshot loop
- companion status sharing
